EVOLUTION OF HTML FROM HTML 2.0 to HTML 5.1

Html has evolved over the years from html 1.0 to the current html 5.1. However, more significantly we would discuss the development from html 2.0 to 5.1.

Versions of HTML

  • HTML 2.0
  • HTML 3.2
  • HTML 4.01
  • HTML 5.1

HTML 2.0

Most webpages often appear the same and boring not having much to offer like using tables, forms and changing background, fonts etc. until html 2.0. Html 2.0 became published in 1995 and had significant improvements. It had all the features in html 1.0 and more features inclusive. This made “HTML 2.0 become the first official set of standards for HTML as the Standard Markup Language for designing and creating websites — the base standard by which all browsers were measured until HTML 3.2.” (Bartels, 2011). It included structural elements for headings, paragraphs, ordered and unordered lists. There was also blockquote for wrapping a quotation from another source. HTML 2.0 had support for forms that allow users to enter and submit data, triggering an action, such as a search. HTML 2.0 focused on making textual content and semantic structure of a resource conveniently readable on different devices and its specification required a String document type declaration beginning with "Doctype HTML PUBLIC”-//IETF?DTD HTML2.0" (Digital preservation, 2018).

HTML 3.2

This version of Html was published January, 1997. It included improved and new features such as form feature to support uploading files and ability to embed a java applet in a rectangular area of a web page. It provided a "DIV" element that could be used to group paragraphs into chapters. It also included extension to support a client-side mechanism for image maps that enable users to access different documents in an image by clicking it. Also, there was a defined Markup for tables. All these gave it more power characteristics for webmasters in designing web pages but these powerful features slowed down the browser (Jasmine, 2018). However, there was no support for representing mathematics, formulae etc. HTML 3.2 specification required that a conforming document start with a document type declaration that begins with "DOCTYPE HTML PUBLIC “-//w3c//DTD HTML 3.2 Final//EN” and not case sensitive (Digital Preservation,2018).

HTML 4.01

This version of HTML was published December, 1999. In addition to the text, multimedia, and hyperlink features of the previous versions of HTML (HTML 3.2 and HTML 2.0), HTML 4.01 supports more multimedia options, scripting languages, style sheets, better printing facilities, and documents that are more accessible to users with disabilities (Digital preservation, 2018). It included cascading style sheets (CSS) which allowed aspects such as text, colour, font and backgrounds to be easily altered. Instead of these aspects being included directly within the webpage, they are now separated, making it much more trouble-free (w3schools, n.d.). HTML 4.01 also takes great strides towards the internationalization of documents, with the goal of making the Web truly World Wide. Since HTML 4 was the current recommendation over a longer period and used much more widely, the description of html 4.01 is minimal. HTML 4.01 improved support for right-to-left and mixed-direction text (aka Bi-Di) and permitted inline text spans to be given a language attribute. The model for tables was extended to serve several objectives, including response to practical experience, support for accessibility, and to simplify import of tables conforming to the widely used CALS Table model. There was Support both for documents constructed from frames (known as framesets) and for inline frames ( known as iframes) (w3.org, n.d.). The "OBJECT" element was introduced as a generalized approach for inserting multimedia. This element was intended to take over from "IMG", "ISMAP", "APPLET", etc. For HTML 4.01, the declaration must begin with "DOCTYPE HTML PUBLIC "--//W3C//DTD HTML 4.01". This string covered Strict, Transitional, and Frameset DTDs.

HTML 5.1

HTML 5.1 is the current version of HTML in use to date. HTML5.1 can be used to write web applications offline when unconnected to the internet; to tell websites where you are physically located; to handle high definition video; and to deliver extraordinary graphics.” (Marshall, 2017). HTML 5.1 continues to evolve and is supported by all of the biggest browsers such as Firefox, Chrome, Safari, Internet Explorer, Opera and Edge (Jasmine Harwood, 2018). Some of the most significant extensions as revealed by Digital preservation, 2018 include:

  • Addition of elements for more explicit representation of document structure, including: "section"; "main"; "article"; "aside"; "header"; "footer"; and "figure".
  • Addition of MathML and SVG markup that can be used inside a document in the HTML serialization, using "math" and "svg" elements.
  • Flexible new elements were introduced for "audio" and "video" to avoid the need for format-specific plug-ins and to provide a consistent set of player functions. The new "canvas" element provided scripts with a rectangular area specified in pixels, to be used for rendering graphs, game graphics, or bit-mapped art on the fly.
  • More effective support for interactivity, for example, to support editing within web pages, with features such as drag-and-drop and spellchecking.
  • Introduction of new HTML APIs made available to developers, particularly for creating interactive applications.
  • The specification that conforms to document to have a document type declaration of "DOCTYPE html", with no case sensitivity.

EVOLUTION OF CSS FROM CSS1 to CSS 3

CSS also evolved over the years from CSS 1 to the current CSS 3. Here is the development.

Versions of CSS

  • CSS 1
  • CSS 2.0
  • CSS 2.1
  • CSS 3

CSS 1

CSS 1(Cascading Style Sheets One) was released 1996 and republished in 1998. Css1 has classified unique groups by attributes which basically include text properties, list properties, border properties and font properties (w3schools.com). It has font properties such as type, size and emphasis on colour of text, background, margin, border, padding, positioning of group attributes, text attributes to include spacing between words, letters and lines of text. It included the alignment of texts, text decoration, text transformation, images, tables and other elements.

CSS 2.0

CSS 2.0 was developed 1999 by W3C to allow authors and users to attach style (e.g. fonts, spacing, bidirectional text and aural cues) to structured documents (e.g. HTML documents and XML applications) by separating the presentation style of document from the content of documents simplifying web authoring and site maintenance (w3.org). It has concepts of media types, new font properties such as shadows. It included absolute, relative and fixed positioning of elements and z-index (Rockcontent, n.d.).

CSS 2.1

CSS2.1 was published June 2011 to fix errors that existed in its previous version. Filter field in the right column enables very fast browsing through CSS2.1 properties, values and selectors (Miloslav Nic, n.d.). it builds on css1 supporting media-specific style sheets so that author may tailor the document presentation to visual browsers, aural devices, printers, braille devices, handheld devices etc. it has support for content positioning, table layout, features for internationalization and some properties related to user interface (AGeek, 2021).

CSS 3

CSS3 is represented in several separate documents called modules. Each module adds new capability or extended features defined in css2 over preserving backward compatibility. There are over 40 css modules published from the css working Group e.g. new selectors, fancy borders, namespace, color, media, multiple backgrounds, box shadows, text shadow, gradient and opacity, vertical text user interaction, speech and advanced animations (Rockcontent, n.d.).
